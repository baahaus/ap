## Design Context

### Users
Blush is for experienced developers who want a terminal coding agent that works efficiently, feels highly customizable, and looks exceptional while they use it. They are opening the CLI to stay in flow, move quickly, and feel like they are using a serious tool with taste instead of a generic shell wrapper.

### Brand Personality
Fun, pretty, minimal. The emotional target is a playful, high-agency developer tool that still feels sharp and efficient. It should feel lively and premium without becoming noisy, toy-like, or slow.

### Aesthetic Direction
The product should borrow:
- Ampcode's motion and interaction confidence
- Flock's crafted product feel
- ap.haus identity and taste

The CLI should lean playful rather than severe, but it should never feel like a novelty terminal toy. It should explicitly avoid the feeling of a boring, stock CLI with plain scrollback and generic text treatment.

### Design Principles
1. Treat the terminal like an application surface, not a log stream.
2. Make motion purposeful: clarify state changes, acknowledge actions, and create a lively feel without delaying work.
3. Keep the interface minimal in structure but rich in detail: small moments, tight hierarchy, refined spacing, strong typography, and selective color.
4. Preserve efficiency for experienced developers: delight must never interfere with speed, legibility, or control.
5. Favor customizable, expressive surfaces over generic defaults, while staying coherent and restrained.
